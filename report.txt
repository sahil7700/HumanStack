Feature 1: Adaptive Checkpoint Plans with Goal Shifting

The core concept: The user sets a goal and a timeframe (e.g. 'get stronger in 12 weeks'). The AI builds a full plan broken into checkpoints every 2-3 weeks. At each checkpoint, the user reviews projected outcomes and can shift what they want — for example, changing from '20% mobility improvement' to '20% strength improvement'. The plan regenerates from that point forward while keeping all completed progress locked.

How it works — user flow
User completes onboarding: goal, timeframe, equipment, constraints
AI generates a 12-week plan split into 4 checkpoints (weeks 3, 6, 9, 12)
Each checkpoint shows: projected outcomes (plain language, not just percentages), actual completion rate, and a Goal Mixer
Goal Mixer: 4 linked sliders (Strength / Endurance / Mobility / Fat Loss) that always sum to 100%
When sliders change, a live preview panel shows what will change in the next 3 weeks
User can write a reason for adjusting (optional, saved to history)
On confirm: weeks from that checkpoint forward regenerate; completed weeks are locked forever
The AI explains every change in plain language: 'Since you shifted toward strength, Tuesdays will now focus on progressive overload instead of circuit work'

Key design rules
Never remove mobility work entirely even if the user weights it at 0 — joint health is non-negotiable
Show checkpoint outcomes as stories, not metrics: 'You should be able to do 3 pull-ups' not 'Upper body strength +18%'
Lock icon on completed weeks — visual confirmation that progress is preserved
History tab: every adjustment the user ever made + what changed as a result

Feature 2: Day-by-Day Session Mode with Live Workout Flow

Every day in the plan gets a full session view. The user taps 'Start Session' and enters a focused, distraction-free mode: exercises appear one at a time, timers run automatically, rest is tracked, and every set they complete is rated so the next one adapts. This is what turns Human Stack from an app you check into an app you actually train with.

Session states (in order)
INTRO — session overview: name, focus, exercise list, estimated time, 'Begin' CTA
EXERCISE VIEW — one exercise at a time: large name, muscle badge, form cue, common mistake warning
SET TIMER — countdown for current set (or rep display). Timer is the biggest element on screen
RPE RATING — after each set: tap 1 of 3: Too Easy / Felt Right / Too Hard
REST TIMER — auto-starts after RPE. Shows next set info. Can skip or extend. Beeps at 0
EXERCISE COMPLETE — brief celebration before next exercise loads
SESSION COMPLETE — total time, sets done, comparison to last week, coach note

Key adaptive rules
RPE 'Too Hard' → reduce next set's reps by 2
RPE 'Too Easy' for 2 sets in a row → suggest adding 5% load next session
If user skips more than 2 rest timers → coach note at end: 'Consider longer rest — it improves output'
Session RPE data feeds back into the checkpoint plan's projections

Feature 3: Smart Onboarding (Calibration Session)
Most fitness apps ask 3 generic questions and generate a plan. Human Stack should feel like talking to a real coach. A 5-minute onboarding flow that asks the right questions — not just 'what's your goal' but 'when did you last train consistently?', 'which movements cause you discomfort?', 'do you prefer morning or evening sessions?' — creates a plan that feels genuinely personal from day one.

The last step of onboarding is a 'Calibration Session' — a short 10-minute session with 5 basic movements (push-up max, bodyweight squat, plank hold, broad jump, and a step test) done at home. The user reports their results. The AI uses these to calibrate starting loads, difficulty, and which exercises to include or exclude. This is the difference between a plan that fits and one that doesn't.

Feature 5: Progressive Overload Auto-Tracking
Progressive overload — gradually increasing the challenge over time — is the single most important principle in fitness. Most apps leave this entirely to the user. Human Stack should handle it automatically: track what the user did last week, apply the right progression rule (beginners: add reps first, then weight; intermediate: wave loading), and show the updated numbers at the start of each set.




PROMPT 5 — Progressive Overload Engine


Build a progressive overload calculation engine in TypeScript.
 
Types:
type SetLog = {
  weight: number | null;   // kg, null for bodyweight
  reps: number;
  rpe: 1 | 2 | 3;         // 1=too easy, 2=felt right, 3=too hard
};
 
type ExerciseHistory = {
  exerciseId: string;
  sessions: Array<{
    date: string;
    sets: SetLog[];
  }>;
};
 
type ProgressionResult = {
  suggestedWeight: number | null;
  suggestedReps: number;
  progressionNote: string;    // plain language explanation shown to user
  progressionType: "add_reps" | "add_weight" | "deload" | "maintain";
};
 
Function: calculateNextSession(
  exercise: Exercise,
  history: ExerciseHistory,
  userTier: "beginner" | "intermediate" | "advanced"
): ProgressionResult
 
Rules:
BEGINNER (< 3 months consistent training):
  - If last session avg RPE <= 2 (too easy): add 1-2 reps, keep weight
  - If reps hit the top of target range AND RPE <= 2: add 2.5kg, reset reps to bottom of range
  - If last session avg RPE >= 3 (too hard): maintain weight, keep reps
  - Never increase weight and reps in the same session
 
INTERMEDIATE (3+ months):
  - Use wave loading: Week 1: high reps/low weight, Week 2: medium, Week 3: low reps/high weight, Week 4: deload
  - Track which week of the wave they are in based on history length % 4
  - If any set had RPE 3 in all sets: do not progress — repeat same session
  
DELOAD (applies to all):
  - Every 4th week: 60% of normal weight, same reps, RPE should be 1 (easy)
  - progressionType = "deload"
 
Output the ProgressionResult with a human-readable progressionNote.
Example: "Last week you hit 10 reps at 40kg feeling good. This week: try 42.5kg for 8 reps."
 
Also write: getWeeklyProgressionSummary(history: ExerciseHistory[]): string
Returns a 2-sentence plain language summary of the user's overall progress trend.
